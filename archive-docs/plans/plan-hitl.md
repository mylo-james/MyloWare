## Core Architecture Pattern

### The n8n Wait Node Pattern

```
Workflow reaches HITL checkpoint
      ↓
Wait Node (On Webhook Call mode)
  - Generates unique resume URL: $execution.resumeUrl
  - Workflow PAUSES here
      ↓
Store resume URL in database (hitl_approvals table)
      ↓
HITL Agent processes user response
      ↓
HITL Agent calls stored resumeUrl
      ↓
Workflow RESUMES from Wait node with approval data
```

**Key Insight**: n8n's Wait node generates a unique webhook URL per execution. External system calls this URL to resume the exact paused execution.

## Phase 1: Standardized HITL Checkpoint Pattern (45 min)

### Pattern to Insert at ANY Workflow Point

This is the reusable pattern that ANY workflow can use:

```
[Previous Node]
      ↓
   [Code Node: Prepare HITL Request]
      ↓
   [HTTP Request: Create Approval + Store Resume URL]
      ↓
   [Wait Node: On Webhook Call]  ← Workflow PAUSES here
      ↓
   [Code Node: Process Approval Response]
      ↓
   [Continue Workflow]
```

### Node 1: Prepare HITL Request (Code Node)

```javascript
// Extract data from previous node
const content = $json;  // What needs approval
const workflowRunId = $('Get Run').item.json.run_id;
const telegramChatId = $('Get Run').item.json.telegram_chat_id;
const stage = 'idea_generation';  // Or dynamic based on context
const projectId = $('Get Run').item.json.project_id;

// Get the resume URL that will be generated by next Wait node
// We'll store this in the database
const resumeUrl = $execution.resumeUrl;

return {
  json: {
    workflowRunId,
    stage,
    content,
    telegramChatId,
    projectId,
    resumeUrl  // Pass to next node
  }
};
```

### Node 2: Create Approval Record (HTTP Request)

- Method: POST
- URL: `https://mcp-vector.mjames.dev/api/hitl/request-approval`
- Body:
  ```json
  {
    "workflowRunId": "{{ $json.workflowRunId }}",
    "stage": "{{ $json.stage }}",
    "content": "{{ $json.content }}",
    "resumeUrl": "{{ $execution.resumeUrl }}",
    "telegramChatId": "{{ $json.telegramChatId }}"
  }
  ```
- Returns: `{ approval: { id, ... } }`

### Node 3: Wait for Approval (Wait Node)

- **Mode**: On Webhook Call
- **Resume On**: Webhook call
- **Respond**: Immediately
- **Authentication**: Header Auth (secure the resume endpoint)

**What happens**:
- n8n generates unique URL: `https://n8n.mjames.dev/webhook-waiting/abc-def-123`
- This URL is available as `$execution.resumeUrl`
- Workflow execution PAUSES here
- When someone calls this URL → workflow resumes

### Node 4: Process Approval Data (Code Node)

```javascript
// Data comes from the webhook call that resumed us
const approvalData = $json;

return {
  json: {
    approved: approvalData.approved,
    feedback: approvalData.feedback,
    selectedItems: approvalData.selectedItems,
    reviewedBy: approvalData.reviewedBy
  }
};
```

### Node 5: Route Decision (If Node)

- Condition: `{{ $json.approved === true }}`
- True → Continue workflow
- False → Handle rejection (retry, notify, etc.)

## Phase 2: Update HITL API to Store Resume URL (30 min)

### Update `src/server/routes/hitl.ts`

Add new endpoint: `POST /api/hitl/request-approval`

```typescript
const requestApprovalSchema = z.object({
  workflowRunId: z.string().uuid(),
  stage: z.enum(['idea_generation', 'screenplay', 'video_generation', 'publishing']),
  content: z.unknown(),
  resumeUrl: z.string().url(),  // NEW: Store the n8n Wait node resume URL
  telegramChatId: z.string().optional(),

app.post('/api/hitl/request-approval', async (request, reply) => {
  const parsed = requestApprovalSchema.safeParse(request.body);
  if (!parsed.success) {
    return sendValidationError(reply, parsed.error);
  }

  // Create approval record WITH resume URL
  const approval = await hitlRepo.createHITLApproval({
    workflowRunId: parsed.data.workflowRunId,
    stage: parsed.data.stage,
    content: parsed.data.content,
    resumeUrl: parsed.data.resumeUrl,  // Store it
  });

  // Trigger Riley to format and send Telegram message
  await executeWorkflow('hitl-review', {
    mode: 'request',
    approvalId: approval.id,
    content: parsed.data.content,
    telegramChatId: parsed.data.telegramChatId,
    stage: parsed.data.stage,
    projectId: parsed.data.projectId,
  });

  return reply.status(201).send({ approval });
});
```

**Triggers**:
1. Execute Workflow Trigger (for both modes - mode determined by params)
2. Telegram Trigger (for inline button + text responses)

**Flow**:

### Branch A: REQUEST Mode (Mode Param = 'request')

**Input from API**:
- `mode: 'request'`
- `approvalId`
- `content` (ideas, screenplay, etc.)
- `telegramChatId`
- `stage`
- `projectId`

**Node: Riley Formats Message (AI Agent)**

System Message:
```
You are Riley, HITL review coordinator.

Identity: persona="hitl-reviewer", project=null

Goal: Format content for human review in Telegram.

Input:
- content: What needs approval (ideas array, screenplay object, etc.)
- stage: Which workflow stage (idea_generation, screenplay, etc.)
- projectId: Project identifier

Task: Create a clear, engaging Telegram message that:

2. Highlights key information
3. Makes it easy to review quickly
4. Uses Telegram markdown formatting

For ideas: Show each idea title and vibe in a numbered list
For screenplay: Show title, logline, and validation summary

Output: Return a formatted message string (max 4096 characters for Telegram).
```

Output Schema:
```json
{
  "type": "object",
  "required": ["message"],
  "properties": {
    "message": {
      "type": "string",
      "description": "Formatted Telegram message with markdown"
    }
  }
}
```

**Node: Send to Telegram**

Telegram Send Message node:
- Chat ID: `{{ $json.telegramChatId }}`
- Text: `{{ $('Riley Formats Message').item.json.message }}`
- Parse Mode: MarkdownV2
- Additional Fields → Reply Markup:
  ```json
  ={{ {
    "inline_keyboard": [[
      {

      },
  ```

### Branch B: RESPONSE Mode (From Telegram Trigger)

**Input from Telegram**:
- `$json.message` (if text) OR
- `$json.callback_query` (if button click)

**Node: Extract Response Data (Code)**

```javascript
const callbackQuery = $json.callback_query;
const message = $json.message;

let responseType, userMessage, approvalId, telegramChatId, decision;

if (callbackQuery) {
  // Button click: "approve:abc-123-def" or "reject:abc-123-def"
  responseType = 'button';
  const [action, id] = callbackQuery.data.split(':');
  approvalId = id;
  decision = action;  // "approve" or "reject"
  userMessage = `User clicked ${action} button`;
  telegramChatId = callbackQuery.message.chat.id;
} else if (message) {
  // Text response
  responseType = 'text';
  userMessage = message.text;
  telegramChatId = message.chat.id;
  
  // Get approvalId from reply context
  // Option 1: User replied to our message, extract from reply_to_message
  // Option 2: Store mapping of chatId → latest approvalId in workflow static data
  // For now: query DB for latest pending approval from this chat
  approvalId = null;  // Will be fetched via API
  decision = null;    // Riley will determine
}

return {
  json: {
    responseType,
    userMessage,
    approvalId,
    telegramChatId,
    decision  // May be null if text response
  }
};
```

**Node: Get Latest Pending Approval (if approvalId is null)**

- If node: Check if `approvalId` is empty
- If empty: HTTP GET `/api/hitl/pending?chatId={{ telegramChatId }}&limit=1`
- Extract approvalId from response

**Node: Get Approval Details**

- HTTP GET: `/api/hitl/approval/{{ approvalId }}`
- Returns: `{ approval: { id, workflowRunId, stage, content, resumeUrl, ... } }`

**Node: Riley Interprets Text (AI Agent - only if decision is null)**

- If node: `{{ !$json.decision }}`
- System Message:
  ```
  You are Riley, HITL review interpreter.
  
  Identity: persona="hitl-reviewer", project=null
  
  Goal: Interpret user feedback to determine approval/rejection.
  
  Input:
  - userMessage: What the user said
  - content: What they're reviewing
  
  Task:
  1. Determine if approving or rejecting
  2. If rejecting, extract specific changes requested
  3. Identify which items to change and how
  
  Examples:
  - "looks good!" → approve
  - "change the third one to be darker" → reject, item: #3, change: "make darker"
  - "velvet puppy should be crystal puppy" → reject, item: "velvet puppy", change: "replace with crystal puppy"
  
  Output: decision, confidence, feedback with specific changes.
  ```

Output Schema:
```json
{
  "type": "object",
  "required": ["decision", "confidence", "feedback"],
  "properties": {
    "decision": {
      "type": "string",
      "enum": ["approve", "reject"]
    },
    "confidence": {
      "type": "number",
      "minimum": 0,
      "maximum": 1
    },
    "feedback": {
      "type": "object",
      "properties": {
        "summary": { "type": "string" },
        "specificChanges": {
          "type": "array",
          "items": {
            "type": "object",
            "properties": {
              "item": { "type": "string" },
              "change": { "type": "string" }
            }
          }
        }
      }
    }
  }
}
```

**Node: Merge Decision (Code)**

```javascript
// Either from button click or AI interpretation
const decision = $json.decision || $('Extract Response Data').item.json.decision;
const feedback = $json.feedback || { summary: 'Approved via button' };

const approvalData = $('Get Approval Details').item.json.approval;

return {
  json: {
    decision,
    feedback,
    approvalId: approvalData.id,
    workflowRunId: approvalData.workflowRunId,
    resumeUrl: approvalData.resumeUrl,  // CRITICAL: URL to resume the paused workflow
    telegramChatId: $('Extract Response Data').item.json.telegramChatId
  }
};
```

**Node: Call Resume URL**

- HTTP Request node
- Method: POST
- URL: `{{ $json.resumeUrl }}`  // The unique Wait node resume URL
- Body:
  ```json
  {
    "approved": "={{ $json.decision === 'approve' }}",
    "feedback": "={{ $json.feedback }}",
    "reviewedBy": "telegram:{{ $json.telegramChatId }}",
    "decision": "={{ $json.decision }}"
  }
  ```

**What happens**: The paused workflow receives this data and continues execution.

**Node: Update HITL Record**

- HTTP Request: `POST /api/hitl/approval/{{ approvalId }}/{{ decision }}`
- Updates database with decision and feedback

**Node: Send Confirmation to Telegram**

- Telegram Send Message
- Text: Based on decision
  - Approve: `✅ Approved! Workflow continuing...`
  - Reject: `❌ Noted. Revising based on your feedback: {{ feedback.summary }}`

## How to Use in ANY Workflow

### Pattern to Copy-Paste

```
[Your workflow reaches checkpoint]
      ↓
[Code: Prepare HITL] → Prepare: workflowRunId, stage, content, resumeUrl
      ↓
[HTTP: Create Approval] → POST /api/hitl/request-approval
      ↓
[Wait: On Webhook Call] → Execution PAUSES, resumeUrl stored in DB
      ↓
      ... (User reviews in Telegram, Riley processes) ...
      ↓
[Wait node resumes] → Receives: { approved, feedback, selectedItems }
      ↓
[Code: Process Response] → Extract approval data
      ↓
[If: Check Approved] → Route based on decision
      ↓
[Continue OR Retry]
```

### To Restart from Any Stage

Call the original workflow with:
```json
{
  "runId": "existing-run-uuid",
  "resumeFromStage": "screenplay",  // Skip to this stage
  "userInput": "original input"
}
```

Workflow checks if runId exists → loads state from DB → jumps to stage.
