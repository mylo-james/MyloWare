## API Overview

### Base URLs and Versioning

Shotstack provides four main APIs, each with production (`v1`) and staging (`stage`) environments:

**Edit API** - Edit videos, images and audio assets using JSON schema and templates
- Base URL: `https://api.shotstack.io/edit/{version}`
- Versions: `v1` (production), `stage` (sandbox/development)

**Serve API** - Inspect and manage hosting of assets generated by Edit and Ingest APIs
- Base URL: `https://api.shotstack.io/serve/{version}`
- Versions: `v1` (production), `stage` (sandbox/development)

**Ingest API** - Upload, store and transform source footage, images, audio and fonts
- Base URL: `https://api.shotstack.io/ingest/{version}`
- Versions: `v1` (production), `stage` (sandbox/development)

**Create API** - Generate images, videos, voice overs and text using generative AI
- Base URL: `https://api.shotstack.io/create/{version}`
- Versions: `v1` (production), `stage` (sandbox/development)

**Quick Reference:**
- Production: `https://api.shotstack.io/v1`
- Sandbox: `https://api.shotstack.io/stage`

## Authentication

All Shotstack API endpoints require authentication using an API key passed in the `x-api-key` header.

```javascript
const headers = {
  'Content-Type': 'application/json',
  'Accept': 'application/json',
  'x-api-key': 'YOUR_API_KEY'
};
```

**Authentication Method:** DeveloperKey (API key)

**Note:** API keys are environment-specific. Use staging keys for `stage` endpoints and production keys for `v1` endpoints.

## API Endpoints Reference

### Edit API Endpoints

#### POST /render
Queue and render the contents of an Edit as a video, image or audio file.

**Base URL:** `https://api.shotstack.io/edit/{version}/render`

**Request Body:** [Edit](#edit-object-structure) JSON object

**Response (201 Created):**
```json
{
  "success": true,
  "message": "Created",
  "response": {
    "message": "Render Successfully Queued",
    "id": "2abd5c11-0f3d-4c6d-ba20-235fc9b8e8b7"
  }
}
```

**JavaScript Example:**
```javascript
const fetch = require('node-fetch');
const editJson = {
  "timeline": { /* ... */ },
  "output": { /* ... */ }
};
const headers = {
  'Content-Type': 'application/json',
  'Accept': 'application/json',
  'x-api-key': 'API_KEY'
};

fetch('https://api.shotstack.io/edit/v1/render', {
  method: 'POST',
  body: JSON.stringify(editJson),
  headers: headers
})
.then(res => res.json())
.then(data => console.log(data));
```

## Error Handling

### HTTP Status Codes

**Success Codes:**
- `200 OK` - Request successful
- `201 Created` - Resource created successfully
- `204 No Content` - Resource deleted successfully

**Client Error Codes:**
- `400 Bad Request` - Validation errors or malformed request
- `401 Unauthorized` - Missing or invalid API key
- `404 Not Found` - Resource not found
- `429 Too Many Requests` - Rate limit exceeded

**Server Error Codes:**
- `500 Internal Server Error` - Server error
- `503 Service Unavailable` - Service temporarily unavailable

### Error Response Format

**Ingest API Error Response:**
```json
{
  "errors": [
    {
      "status": "400",
      "title": "Validation Error",
      "detail": "\"url\" is required"
    }
  ]
}
```

**Generated Asset Error Response:**
```json
{
  "errors": [
    {
      "status": 400,
      "title": "Bad Request",
      "detail": "\"body/options/text\" is required."
    }
  ]
}
```

**JavaScript Error Handling:**
```javascript
fetch('https://api.shotstack.io/edit/v1/render', {
  method: 'POST',
  body: JSON.stringify(editJson),
  headers: headers
})
.then(async (res) => {
  if (!res.ok) {
    const error = await res.json();
    throw new Error(`API Error: ${error.errors?.[0]?.detail || res.statusText}`);
  }
  return res.json();
})
.then(data => console.log('Success:', data))
.catch(error => console.error('Error:', error));
```

### Render Status Values

Possible values for render `status`:
- `queued` - Render has been queued
- `rendering` - Render is in progress
- `done` - Render completed successfully
- `failed` - Render failed

**Polling Example:**
```javascript
async function pollRenderStatus(renderId, maxAttempts = 60) {
  for (let i = 0; i < maxAttempts; i++) {
    const response = await fetch(
      `https://api.shotstack.io/edit/v1/render/${renderId}`,
      { headers: { 'x-api-key': 'API_KEY', 'Accept': 'application/json' } }
    );
    const data = await response.json();
    
    if (data.response.status === 'done') {
      return data.response.url;
    } else if (data.response.status === 'failed') {
      throw new Error('Render failed');
    }
    
    // Wait 2 seconds before next poll
    await new Promise(resolve => setTimeout(resolve, 2000));
  }
  throw new Error('Render timeout');
}
```

### Configuring Webhooks

Instead of polling for render status, use webhooks for real-time notifications. Add a `callback` URL to your render request:

```json

  "output": { /* ... */ },
  "callback": "https://my-application.com/webhook.php"
}
```

```javascript
// Listen for clip selection events
```

### Shotstack Studio SDK Advanced Features

Source: https://shotstack.io/docs/api/guide/studio-sdk

The Studio SDK provides advanced editing capabilities beyond basic initialization.

**Playback Controls:**

```javascript
// Playback controls
edit.play();
edit.pause();
edit.seek(2000); // Seek to 2 seconds (in milliseconds)
edit.stop(); // Stop and return to beginning
```

**Editing Operations:**

```javascript
// Add clip to track
edit.addClip(0, {
  asset: {
    type: "image",
    src: "https://example.com/image.jpg",
  },
  start: 0,
  length: 5,
});

// Add new track
edit.addTrack(1, { clips: [] });

// Delete clip
edit.deleteClip(0, 0);

// Delete track
edit.deleteTrack(1);
```

**Retrieve Edit Information:**

```javascript
// Get clip
const clip = edit.getClip(0, 0);

// Get track
const track = edit.getTrack(0);

// Get complete edit JSON
const editJson = edit.getEdit();

// Get total duration
const duration = edit.totalDuration; // in milliseconds
```

**Event Handling:**

```javascript
// Listen for clip selection events
edit.events.on("clip:selected", (data) => {
  console.log("Clip selected:", data.clip);
  console.log("Track index:", data.trackIndex);
  console.log("Clip index:", data.clipIndex);
});

// Listen for clip update events
edit.events.on("clip:updated", (data) => {
  console.log("Previous state:", data.previous);
  console.log("Current state:", data.current);
});
```
