### Quick Start with Shotstack Studio SDK

Source: https://shotstack.io/docs/api/guide/studio-sdk

A rapid introduction to setting up and using the Shotstack Studio SDK. This snippet demonstrates how to fetch a video template, initialize the core 'Edit' and 'Canvas' components, load the template for editing, and enable keyboard controls for interactive manipulation.

```javascript
import { Edit, Canvas, Controls } from "@shotstack/shotstack-studio";

// 1. Retrieve an edit from a template
const templateUrl =
  "https://shotstack-assets.s3.amazonaws.com/templates/hello-world/hello.json";
const response = await fetch(templateUrl);
const template = await response.json();

// 2. Initialize the edit with dimensions and background color
const edit = new Edit(template.output.size, template.timeline.background);
await edit.load();

// 3. Create a canvas to display the edit
const canvas = new Canvas(template.output.size, edit);
await canvas.load(); // Renders to [data-shotstack-studio] element

// 4. Load the template
await edit.loadEdit(template);

// 5. Add keyboard controls
const controls = new Controls(edit);
await controls.load();
```

Comprehensive documentation for the Shotstack API's `POST /assets` endpoint, which is used to generate new assets like text-to-speech. This section details the request body structure, parameters, example responses, and error codes.

```APIDOC
Endpoint: POST /assets
Description: Generate an asset using a Shotstack or third party provider. Choose the provider, type of asset and provide prompts, inputs and options to generate the asset. Once generated the asset can be used with the Edit API or downloaded.
Base URL: https://api.shotstack.io/create/{version}

Request Body Example:
{
  "provider": "shotstack",
  "options": {
    "type": "text-to-speech",
    "text": "This is a text to speech example generated by Shotstack",
    "voice": "Matthew",
    "language": "en-US",
    "newscaster": false
  }
}

Parameters:
- Name: body
  In: body
  Type: [GeneratedAsset](#schemageneratedasset)
  Required: true
  Description: Generate an asset using the specified provider, type and options.

Example Responses:
201 Response:
{
  "data": {
    "type": "asset",
    "id": "01gz0-tj679-xj30t-hr8zk-3hasvk",
    "attributes": {
      "owner": "5ca6hu7s9k",
      "provider": "shotstack",
      "type": "text-to-speech",

      "status": "done",
      "created": "2023-01-02T01:47:18.973Z",
      "updated": "2023-01-02T01:47:37.260Z"
    }
  }
}

Responses:
- Status: 201
  Meaning: Created (https://tools.ietf.org/html/rfc7231#section-6.3.2)
  Description: The generated asset details
  Schema: [GeneratedAssetResponse](#schemageneratedassetresponse)
- Status: 400
  Meaning: Bad Request (https://tools.ietf.org/html/rfc7231#section-6.5.1)
  Description: A list of validation and other errors
  Schema: [GeneratedAssetErrorResponse](#schemageneratedasseterrorresponse)

Authentication:
To perform this operation, you must be authenticated by means of one of the following methods:
DeveloperKey
```

POST /upload
Request a signed URL to upload a file to. The response returns a signed URL that you use to upload the file to.
The signed URL looks similar to:
<https://shotstack-ingest-api-stage-sources.s3.ap-southeast-2.amazonaws.com/5ca6hu7s9k/zzytey4v-32km-kq1z-aftr-3kcuqi0brad2/source?AWSAccessKeyId=ASIAWJV7UWDMGTZLHTXP&Expires=1677209777&Signature=PKR4dGDDdOuMTAQmDASzLGmLOeo%3D&x-amz-acl=public-read&x-amz-security-token=IQoJb3JpZ2luX2VjEGMaDmFwLX......56osBGByztm7WZdbmXzO09KR>
In a separate API call, use this signed URL to send a PUT request with the binary file. Using cURL you can use
a command like:
`curl -X PUT -T video.mp4 {data.attributes.url}`

### Shotstack API Base URLs and Versioning

Source: https://shotstack.io/docs/api/api

This section details the base URLs for the Shotstack Edit, Serve, Ingest, and Create APIs. It also describes the 'version' parameter, which allows switching between production (v1) and development (stage) environments for each API.

```APIDOC
Edit API:
  Description: Edit videos, images and audio assets in the cloud using a simple JSON schema and templates.
  Base URL: https://api.shotstack.io/edit/{version}
  Parameters:
    version:
      Description: Set the stage to v1 for production usage without watermarks. Set to stage to use the development sandbox.
      Type: string
      Possible Values: v1, stage
      Default: v1

Serve API:
  Description: Inspect and manage the hosting of assets generated by the Edit and Ingest APIs.
  Base URL: https://api.shotstack.io/serve/{version}
  Parameters:
    version:
      Description: Set the stage to v1 for production usage. Set to stage to use the development sandbox.
      Type: string
      Possible Values: v1, stage
      Default: v1

Ingest API:
  Description: Ingest (upload, store and transform) source footage, images, audio and fonts to be used by the Edit API.
  Base URL: https://api.shotstack.io/ingest/{version}
  Parameters:
    version:
      Description: Set the stage to v1 for production usage. Set to stage to use the development sandbox.
      Type: string
      Possible Values: v1, stage
      Default: v1
```

### Shotstack Clip Properties

Source: https://shotstack.io/docs/api/api

Defines the configurable properties for a Clip object in the Shotstack video editing API, including asset source, start and length timings, visual fit, scale, position, offset, transitions, effects, filters, and opacity. Some properties support multiple types (e.g., number or string for timing) with specific behaviors.

```APIDOC
Clip Properties:
  - asset:
      Type: [Asset](#schemaasset)
      Required: true
      Restrictions: none
      Description: The type of asset to display for the duration of the Clip, i.e. a video clip or an image. Choose from one of the available asset types below.
  - start:
      Type: any (oneOf: number, string)
      Required: true
      Restrictions: none
      Description: The start position of the Clip on the timeline.
        - If number: The start position of the Clip on the timeline, in seconds. For example, to start the Clip at 2 seconds, set the start value to 2.
        - If string: The start position using a [smart clip property](/docs/guide/architecting-an-application/smart-clips/). Set to `auto` to automatically play the clip after the previous clip finishes.
  - length:
      Type: any (oneOf: number, string)
      Required: true
      Restrictions: none
      Description: The duration the Clip should play for.
        - If number: The duration the Clip should play for, in seconds. For example, to play the Clip for 5 seconds, set the length value to 5.
        - If string: The duration the Clip should play for using a [smart clip property](/docs/guide/architecting-an-application/smart-clips/). Set to `auto` to play the Clip for the duration of the asset. Set to `end` to display or play the clip to the end of the timeline.
  - fit:
      Type: string
      Required: false
      Restrictions: none
      Description: Set how the asset should be scaled to fit the viewport using one of the following options:
        * `crop` **(default)** - scale the asset to fill the viewport while maintaining the aspect ratio. The asset will be cropped if it exceeds the bounds of the viewport.
        * `cover` - stretch the asset to fill the viewport without maintaining the aspect ratio.
        * `contain` - fit the entire asset within the viewport while maintaining the original aspect ratio.
        * `none` - preserves the original asset dimensions and does not apply any scaling.
  - scale:
      Type: number
      Required: false
      Restrictions: none
      Description: Scale the asset to a fraction of the viewport size - i.e. setting the scale to 0.5 will scale asset to half the size of the viewport. This is useful for picture-in-picture video and scaling images such as logos and watermarks.
  - position:
      Type: string
      Required: false
      Restrictions: none
      Description: Place the asset in one of nine predefined positions of the viewport. This is most effective for when the asset is scaled and you want to position the element to a specific position.
        * `top` - top (center)
        * `topRight` - top right
        * `right` - right (center)
        * `bottomRight` - bottom right
        * `bottom` - bottom (center)
        * `bottomLeft` - bottom left
        * `left` - left (center)
        * `topLeft` - top left
        * `center` - center
  - offset:
      Type: [Offset](#schemaoffset)
      Required: false
      Restrictions: none
      Description: Offset the location of the asset relative to its position on the viewport. The offset distance is relative to the width of the viewport - for example an x offset of 0.5 will move the asset half the viewport width to the right.
  - transition:
      Type: [Transition](#schematransition)
      Required: false
      Restrictions: none
      Description: In and out transitions for a clip - i.e. fade in and fade out
  - effect:
      Type: string
      Required: false
      Restrictions: none
      Description: A motion effect to apply to the Clip.
        * `zoomIn` - slow zoom in
        * `zoomOut` - slow zoom out
        * `slideLeft` - slow slide (pan) left
        * `slideRight` - slow slide (pan) right
        * `slideUp` - slow slide (pan) up
        * `slideDown` - slow slide (pan) down
        The motion effect speed can also be controlled by appending `Fast` or `Slow` to the effect, e.g. `zoomInFast` or `slideRightSlow`.
  - filter:
      Type: string
      Required: false
      Restrictions: none
      Description: A filter effect to apply to the Clip.
        * `blur` - blur the scene
        * `boost` - boost contrast and saturation
        * `contrast` - increase contrast
        * `darken` - darken the scene
        * `greyscale` - remove colour
        * `lighten` - lighten the scene
        * `muted` - reduce saturation and contrast
        * `negative` - negative colors
  - opacity:
      Type: any (oneOf: number, array of Tween objects)
      Required: false
      Restrictions: none
      Description: Offset an asset on the horizontal axis (left or right). Use a number or an array of [Tween](./#tocs_tween) objects to create a custom animation.
        - If number: Sets the opacity of the Clip where 1 is opaque and 0 is transparent.
```

### Shotstack Media Editing API Workflow

Source: https://shotstack.io/docs/api/guide/architecting-an-application/guidelines

This snippet details the sequential API interactions required to edit and render media using Shotstack, from defining the edit to retrieving the final output. It covers the preparation of JSON, initiating the render, monitoring its progress, and handling the resulting media file.

```APIDOC
Shotstack Media Editing Workflow:

1. Prepare JSON using the Studio:
   - Action: Define the video edit using JSON, specifying asset placement and timing on a timeline.
   - Tool: Shotstack Studio is recommended for creating and validating the JSON data.

2. Post JSON to API Render Endpoint:
   - Method: POST
   - Endpoint: /render
   - Input: The prepared JSON data representing the media edit.
   - Purpose: Initiates the rendering process on Shotstack's servers.

3. Poll Render Endpoint:
   - Method: GET
   - Endpoint: /render/{id} (where {id} is the render ID returned from the POST request)
   - Purpose: Periodically check the status of the video render.
   - Status: Look for 'done' to confirm completion.
   - Note: Webhooks are highly recommended as an alternative to polling for real-time notifications upon render completion, reducing API calls.

4. Hosting and Serving Video:
   - Availability: Once rendering is complete, the video is accessible via a temporary URL.
   - Storage: Automatically transferred to the Shotstack hosting service.

5. Download or Transfer the Video:
   - Action: Fetch the rendered video from Shotstack's servers (S3).
   - Options: Transfer to your own S3 bucket, Google Drive, Vimeo, or other desired destinations.
   - Important: If not using Shotstack's hosting, videos are stored on Shotstack servers for 24 hours only before deletion.
```

### Shotstack Text-to-Image Options API Definition

Source: https://shotstack.io/docs/api/api

Defines the options for the Shotstack text-to-image service. Users can set a text prompt to generate an image, which will be output as a PNG file. The response will include a URL to the generated image.

```json
{
  "type": "text-to-image",
  "prompt": "A detailed photograph of Mars, showcasing its orange-red surface",
  "width": 512,
  "height": 512
}
```

```APIDOC
ShotstackTextToImageOptions:
  Properties:
    type: string (Required: true) - The type of asset to generate - set to `text-to-image` for text-to-image.
    prompt: string (Required: true) - The text prompt to generate an image from.
    width: integer (Required: true) - The width of the image in pixels.
    height: integer (Required: true) - The height of the image in pixels.
  Enumerated Values:
    type: text-to-image
```
